## 1. 环境准备

### 1.1 检查开发环境

#### 必需软件
- **DevEco Studio**: 6.0.2 Release
- **Node.js**: 16.x 或更高版本
- **HarmonyOS SDK**: API 9 或更高版本
- **JDK**: 11 或更高版本(DevEco Studio自带)

#### 检查步骤

1. **检查DevEco Studio版本**
   - 打开DevEco Studio
   - 点击 `Help` → `About`
   - 确认版本为 6.0.2 Release

2. **检查HarmonyOS SDK**
   - 打开DevEco Studio
   - 点击 `File` → `Settings` → `SDK`
   - 在 `HarmonyOS` 标签页中确认已安装API 9或更高版本

3. **检查Node.js版本**
   - 打开终端
   - 运行命令: `node -v`
   - 确认版本为16.x或更高

### 1.2 安装HarmonyOS SDK

如果尚未安装HarmonyOS SDK,请按以下步骤安装:

1. 打开DevEco Studio
2. 点击 `File` → `Settings` → `SDK`
3. 点击 `HarmonyOS` 标签页
4. 点击 `Edit` 按钮
5. 勾选需要的API版本(至少API 9)
6. 点击 `Apply` 和 `OK` 开始下载安装

## 3. 配置项目

### 3.1 配置Gradle

#### 3.1.1 配置本地Gradle

1. 打开 `File` → `Settings` → `Build, Execution, Deployment` → `Build Tools` → `Gradle`
2. 选择 `Use Gradle from`: `Specified location`
3. 设置Gradle home路径(如果需要)

#### 3.1.2 配置Gradle仓库

编辑项目根目录下的 `hvigorfile.ts`:

```typescript
import { hapTasks } from '@ohos/hvigor-ohos-plugin'

export default {
    system: hapTasks,  /* Built-in plugin of Hvigor. It cannot be modified. */
    plugins:[]         /* Custom plugin to extend the functionality of Hvigor. */
}
```

### 3.2 配置签名

#### 3.2.1 创建调试签名

1. 点击 `File` → `Project Structure`
2. 选择 `Project` → `Signing Configs`
3. 勾选 `Automatically generate signature`
4. 点击 `OK`

或者手动配置:

1. 点击 `File` → `Project Structure`
2. 选择 `Project` → `Signing Configs`
3. 填写签名信息:
   - `Store File`: 选择或创建.keystore文件
   - `Store Password`: 输入密码
   - `Key Alias`: 输入别名
   - `Key Password`: 输入密码

#### 3.2.2 配置自动签名(推荐)

1. 打开 `entry` 模块的 `build-profile.json5`
2. 确认包含签名配置:

```json5
{
  "apiType": "stageMode",
  "buildOption": {},
  "targets": [
    {
      "name": "default",
      "runtimeOS": "HarmonyOS"
    }
  ]
}
```

### 3.3 配置模拟器

#### 3.3.1 创建模拟器

1. 点击工具栏的设备下拉菜单
2. 选择 `Device Manager`
3. 点击 `+` 创建新模拟器
4. 选择设备类型(Phone或Tablet)
5. 选择系统镜像(API 9或更高)
6. 点击 `Next` 和 `Finish`

#### 3.3.2 启动模拟器

1. 在Device Manager中选择创建的模拟器
2. 点击 `Play` 按钮启动模拟器

## 4. 安装依赖

### 4.1 安装项目依赖

#### 方法一: 使用DevEco Studio

1. 打开终端窗口(`View` → `Tool Windows` → `Terminal`)
2. 运行命令:
   ```bash
   npm install
   ```

#### 方法二: 使用命令行

在项目根目录下运行:

```bash
cd C:\Users\mingy\Desktop\lx-music\harmony-music-skeleton
npm install
```

### 4.2 验证依赖安装

安装完成后,检查 `oh_modules` 目录是否存在:

```bash
ls oh_modules
```

应该能看到安装的依赖包。

### 4.3 解决依赖问题

如果遇到依赖安装问题:

#### 问题1: Node.js版本不兼容

```bash
# 更新Node.js到16.x或更高版本
# 使用nvm管理Node.js版本
nvm install 16
nvm use 16
```

#### 问题2: 网络问题

使用国内镜像源:

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

#### 问题3: 权限问题(Windows)

以管理员身份运行DevEco Studio或命令行。

## 5. 运行项目

### 5.1 选择运行设备

1. 在工具栏的设备下拉菜单中选择设备
   - 模拟器: 选择已启动的模拟器
   - 真机: 连接HarmonyOS设备并启用USB调试

### 5.2 构建项目

#### 方法一: 使用工具栏

1. 点击工具栏的 `Run` 按钮(绿色三角形)
2. 或使用快捷键 `Shift + F10`

#### 方法二: 使用Gradle任务

1. 打开 `View` → `Tool Windows` → `Gradle`
2. 展开 `harmony-music-skeleton` → `Tasks`
3. 双击 `build` 或 `assembleHap`

### 5.3 查看运行日志

#### 查看应用日志

1. 打开 `View` → `Tool Windows` → `HiLog`
2. 过滤日志级别(Info、Debug、Error等)
3. 搜索关键字(如"PlayerService"、"MusicService")查看特定日志

#### 常用日志过滤

```bash
# 查看所有日志
*

# 查看错误日志
*:E

# 查看特定服务的日志
PlayerService*
MusicService*

# 查看应用启动日志
EntryAbility*
```

### 5.4 调试项目

#### 启动调试模式

1. 点击工具栏的 `Debug` 按钮(虫子图标)
2. 或使用快捷键 `Shift + F9`

#### 设置断点

1. 在代码行号左侧点击,设置断点
2. 断点会显示为红色圆点

#### 调试操作

- `F8`: Step Over(单步执行)
- `F7`: Step Into(进入方法)
- `Shift + F8`: Step Out(跳出方法)
- `F9`: Resume(继续执行)

## 6. 常见问题

### 6.1 构建错误

#### 错误1: SDK版本不兼容

**错误信息**:
```
API version is not supported
```

**解决方案**:
1. 打开 `File` → `Settings` → `SDK`
2. 安装需要的API版本
3. 修改 `build-profile.json5` 中的API版本

#### 错误2: 依赖缺失

**错误信息**:
```
Module not found: Error: Can't resolve '@ohos/axios'
```

**解决方案**:
```bash
# 重新安装依赖
rm -rf oh_modules
npm install
```

#### 错误3: 签名配置错误

**错误信息**:
```
Signature configuration error
```

**解决方案**:
1. 检查签名配置是否正确
2. 重新配置签名信息
3. 使用自动签名功能

### 6.2 运行时错误

#### 错误1: 应用崩溃

**错误信息**:
```
Application crashed
```

**解决方案**:
1. 查看HiLog日志
2. 检查代码逻辑错误
3. 验证权限配置

#### 错误2: 权限被拒绝

**错误信息**:
```
Permission denied
```

**解决方案**:
1. 检查 `module.json5` 中的权限配置
2. 确保已申请必要的权限
3. 在运行时请求权限

### 6.3 性能问题

#### 问题1: 应用启动慢

**解决方案**:
1. 优化初始化逻辑
2. 使用延迟加载
3. 减少启动时的网络请求

#### 问题2: 内存占用高

**解决方案**:
1. 检查内存泄漏
2. 及时释放资源
3. 优化图片加载

### 6.4 开发工具问题

#### 问题1: DevEco Studio卡顿

**解决方案**:
1. 增加DevEco Studio内存:
   - 编辑 `DevEco Studio64.exe.vmoptions`
   - 调整 `-Xmx` 参数
2. 关闭不必要的插件
3. 清理缓存: `File` → `Invalidate Caches`

#### 问题2: 代码提示不工作

**解决方案**:
1. 重新索引项目: `File` → `Invalidate Caches`
2. 检查SDK配置
3. 重启DevEco Studio

## 7. 快速开始

### 最简启动步骤

1. **打开DevEco Studio 6.0.2**
2. **打开项目**: `File` → `Open` → 选择项目目录
3. **安装依赖**: 在终端运行 `npm install`
4. **选择设备**: 在工具栏选择模拟器或连接真机
5. **运行项目**: 点击 `Run` 按钮(Shift + F10)

### 验证项目运行成功

应用启动后,您应该能看到:
- 首页显示"欢迎使用音乐播放器"
- 可以点击"搜索音乐"跳转到搜索页
- 可以点击"播放器"跳转到播放器页(当有音乐播放时)
